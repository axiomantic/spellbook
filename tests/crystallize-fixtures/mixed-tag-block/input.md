# Example SOP — Mixed Tag Block

This SOP contains a `<CRITICAL>` block that mixes an imperative rule
paragraph with a narrative rationale paragraph. Per design item #9.2 the
block must be split at the first IMPERATIVE-to-NARRATIVE paragraph
boundary: the imperative paragraph is lifted (with `<CRITICAL>` wrapping
preserved) into the canonical Rules section, and the narrative tail stays
in place, unwrapped, as General Instructions content.

## Phase 4

<CRITICAL>
NEVER skip Phase 4.5 iteration loop. Maximum 3 iterations.

This rule exists because skipping iteration produced silent capability loss
in three documented cases (skill-A, skill-B, skill-C). The iteration loop is
the safety net for compression mistakes.
</CRITICAL>

## Closing

The split must produce two outputs whose concatenation accounts for all
original content with no duplication and no loss.
